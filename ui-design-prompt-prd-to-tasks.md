# UI Design Prompt: PRD â†’ ERD â†’ Tasks Converter

## Overview

Design a modern, professional web application UI for a tool that converts Product Requirements Documents (PRD) into Entity-Relationship Diagrams (ERD) and detailed development tasks. The tool is aimed at Business Analysts, Solution Architects, and non-developers who need to create technical specifications without deep coding knowledge.

## Design Requirements

### Visual Style
- **Theme:** Clean, modern, professional (similar to Notion, Linear, or Figma)
- **Color scheme:** Light mode primary with dark mode support
- **Primary color:** Blue (#3B82F6) or similar professional blue
- **Accent colors:** 
  - Success/Complete: Green (#10B981)
  - Warning: Amber (#F59E0B)
  - Error/Critical: Red (#EF4444)
  - Info: Blue (#3B82F6)
- **Typography:** Inter or similar clean sans-serif font
- **Spacing:** Generous whitespace, 8px grid system
- **Border radius:** Subtle rounding (6-8px for cards, 4px for inputs)
- **Shadows:** Subtle, layered shadows for depth

### Responsive Behavior
- Minimum supported width: 1280px (desktop-focused tool)
- Panels should be resizable via drag handles
- Left and right panels should be collapsible
- Collapsed state shows icons only with tooltips

## Keyboard Shortcuts

| Shortcut | Action |
|----------|--------|
| Ctrl/Cmd + S | Save project |
| Ctrl/Cmd + O | Open project |
| Ctrl/Cmd + E | Export tasks |
| Ctrl/Cmd + 1-4 | Jump to phase 1-4 |
| Ctrl/Cmd + [ | Collapse left panel |
| Ctrl/Cmd + ] | Collapse right panel |
| Ctrl/Cmd + , | Open settings |
| Esc | Close modals |

---

## Responsive Behavior

### Minimum Width (1280px)
- All three panels visible
- Panels at minimum widths

### Comfortable Width (1600px+)
- Generous panel sizes
- Full content visible without scrolling

### Panel Collapse Triggers
- Below 1280px: Auto-collapse right panel
- Below 1024px: Auto-collapse both side panels
- Manual toggle always available

---

## Component Library Suggestions

Recommend using these components/libraries:
- **UI Framework:** shadcn/ui (Radix-based, customizable)
- **ERD Canvas:** ReactFlow
- **Code Editor:** Monaco Editor (for PRD editing)
- **Markdown Preview:** react-markdown with remark-gfm
- **Icons:** Lucide React
- **Drag & Drop:** @dnd-kit or react-beautiful-dnd
- **Resizable Panels:** react-resizable-panels
- **Toast Notifications:** sonner or react-hot-toast
- **State Management:** Zustand

## Summary

This UI design provides:

1. **Three-panel layout** for clear separation of concerns
2. **Progressive workflow** via phase stepper
3. **Real-time analysis feedback** in right panel
4. **Flexible document preview** with source/preview toggle
5. **Comprehensive settings** for API keys and model selection
6. **Visual ERD editing** with ReactFlow
7. **Export options** for multiple formats

The design prioritizes:
- Clarity and focus on the current task
- Non-developer friendliness
- Professional appearance
- Efficient workflows with keyboard shortcuts
- Responsive behavior for different screen sizes
